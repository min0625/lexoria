# 畫面清單與 UI 流程

> 主設計文件的配套文件（見 [docs/word-puzzle-game-design.md](docs/word-puzzle-game-design.md)），只管「有哪些畫面、長什麼樣、怎麼切換」。UX 原則（回饋動畫、觸控細節）在主文件 §4、§7，此處不重複。

## 1. 畫面清單

整個遊戲只有 **2 個全螢幕畫面 + 5 個 overlay**。沒有標題畫面——開啟即進入上次進行中的關卡（Wordscapes 同款做法；品牌露出交給 Phase 2 的 native 啟動畫面）。

| # | 名稱 | 型態 | 進入方式 |
|---|------|------|---------|
| A | 遊戲主畫面 | 全螢幕 | 開啟預設；從關卡選擇點關卡 |
| B | 關卡選擇 | 全螢幕 | 主畫面點左上關卡編號 |
| C | 過關 | overlay | 找齊全部目標字時自動 |
| D | 設定 | overlay | 主畫面點齒輪 |
| E | 查詞卡片 | popover | 點已找到的目標字格 |
| F | 教學 | overlay | 第 1 關首次進入自動；完成第一個字即收 |
| G | 全破 | 全螢幕 | 最後一關過關後的「下一關」 |
| H | 載入失敗 | 全螢幕 | 開啟時 levels.json 取不到（斷網/部署中）自動；僅一句訊息＋重新載入按鈕 |
| — | 橫式遮罩 | overlay | 偵測到橫向時自動 |

**刻意不做的畫面**：標題/主選單、每日獎勵、商店、成就——Phase 3 以後的事（主文件 §16）。

## 2. 畫面 A：遊戲主畫面

```
┌──────────────────────────┐
│ [第42關] 🪙128 [+25] ⚙  │ ← 頂列：關卡鈕(進B)、金幣、定時領取鈕(可領時顯示+25；冷卻中壓暗顯示 h:mm 倒數，仍可點——點了抖動並顯示「X 小時 Y 分後可領取金幣」，避免倒數被誤讀成限時。最後一分鐘改逐秒顯示「59 秒」，此時點擊訊息改「就快好了！」；歸零翻成+25 的瞬間播彈跳動畫)、設定鈕(開D)
│                          │
│     ┌─┬─┬─┬─┬─┐         │
│     │P│L│A│N│E│         │
│     └─┼─┴─┴─┴─┘         │ ← 填字格：置中，佔上方約 55% 高
│       │E│                │
│       │A│ ...            │
│                          │
│        ｢ P L A ｣         │ ← 目前拼字串：僅手勢進行中顯示
│                          │
│  💡        （A）          │
│  -25    （P）🔀（E）      │ ← 字母轉盤：下 1/3 拇指區，
│        （L）（N）         │    中央洗牌鈕
```

- 頂列高度固定，格子區域用剩餘空間置中縮放（格子尺寸 = 可用寬高取小除以行列數，主文件 §7）。
- 提示鈕（💡 −25）固定在轉盤左側；金幣不足時按下 → 鈕抖動並強調價格（主文件 §8）。
- 拼字串顯示區在格子與轉盤之間，平時空白、不佔版面跳動（高度預留固定）。

## 3. 畫面 B：關卡選擇

```
┌──────────────────────────┐
│ ← 返回          🪙128    │
│                          │
│  ✓1  ✓2  ✓3  ✓4  ✓5     │
│  ✓6 ...  ▶42  🔒43 🔒44  │ ← 網格列表：✓已完成(可重玩)
│  🔒45 ...                │    ▶目前關 🔒未解鎖
└──────────────────────────┘
```

- 純網格按鈕列表，捲動即可，不需要分頁或章節地圖；進入時自動捲到目前關卡（置中）。
- 重玩已完成關卡：進入時 `levelState` 重新開始，但不再給金幣（主文件 §7）。重玩中離開再回來 → 回到 `currentLevel`，不保存重玩進度（`levelState` 只屬於 `currentLevel`，存檔結構不用改）。

## 4. Overlay 們

**C 過關**：半透明遮罩 + 中央卡片：「過關！ +10 🪙」、本關找到的 bonus 數、本關全部目標字（可點，彈出 E 查詞卡）、［下一關］按鈕。點卡片外不關閉（過關是單向流程）。最後一關的［下一關］→ 進畫面 G。列出目標字是為了讓玩家能查完成最後一字時來不及點格子看到的釋義。

**D 設定**：半透明遮罩 + 卡片：音效開關、震動開關、［分享］按鈕（純文字：emoji 格盤 + 關卡數 + 字母盤。格盤只畫關卡形狀、不畫進度，進度由關卡數表達；字母隨文案附上讓收到的人能直接在聊天室裡解題（與下載圖同樣的取捨——字母本來就是公開輸入，答案才是雷）。字數與方向不必寫進文案：每段長度 ≥2 的連續白格直線即一個字，這條填字慣例足以從盤面推回全部字位（已對 500 關驗證無例外）（⬜ 可填的格／⬛ 背景，沿用填字遊戲「白格可下筆、黑格是牆」的通用慣例而非品牌色；兩者同屬 U+2B1x 區塊以確保每列對齊，理由詳見 `snapshotText` 註解）；桌機手機一律先複製到剪貼簿再開系統分享面板——部分分享目標（桌面面板、Facebook）只收連結、丟掉文字，剪貼簿裡永遠有完整版可自行貼上；帶檔分享時多數目標也會丟文字，故一律不帶圖）、［下載圖片］按鈕（canvas 快照 PNG 直接下載；目前以 `hidden` 隱藏，程式碼保留備用）、「關於」（版本號 + 素材授權聲明，主文件 §14 要求的授權頁就放這）。點卡片外關閉。

**E 查詞卡片**：貼著被點的格子彈出的小 popover：單字、一句繁中釋義（主）、一句英文釋義（輔）、🔊 喇叭鈕（Web Speech API，主文件 §6）。點卡片外任意處關閉。無英文語音可用時隱藏喇叭鈕。

**F 教學**：第 1 關疊在轉盤上的手指軌跡動畫 + 一句文案，不攔截操作——玩家照著滑或自己亂滑都行，完成第一個字即永久收掉（存檔記 `tutorialDone`，併入 `settings` 旁即可）。

**G 全破**：全螢幕「更多關卡即將推出」+ 返回關卡選擇按鈕（可重玩舊關）。

**橫式遮罩**：全螢幕不透明「請轉直畫面」，蓋過其他一切。

### 疊層順序（z-index 由上到下）

```
橫式遮罩 > 查詞卡(E) > 過關(C)/全破(G) > 設定(D) > 教學(F) > 遊戲主畫面
```

同時只會出現一個可互動 overlay，唯一例外是 E：E 可疊在 C 之上（點過關卡片列出的目標字彈出），因為 C 是單向流程、不會被 E 誤觸關閉。開 D 前仍先關 E；C 觸發時只重置舊的 E（避免錨點指向已消失的格子），不強制關閉、C 內可再點字重新彈出。教學(F)不攔截操作所以不算互動層。

## 5. 導覽流程

```
（開啟）→ A 遊戲主畫面 ⇄ B 關卡選擇
              │ 找齊目標字
              ▼
           C 過關 ──下一關──▶ A（下一關）
              └─（最後一關）──▶ G 全破 ──▶ B
```

- 沒有瀏覽器返回鍵整合：Phase 1 是單頁 Web，不動 history API；Capacitor 包殼後再處理 Android 返回鍵（B→A、overlay→關閉）。
- 畫面切換 = 顯示/隱藏兩個 `<section>`，不做路由。

## 7. 色票草案（主文件 §19-3 的拍板素材）

給一版可以直接開工的預設值；視覺定案時只改這組變數，元件不動。
只做深色一套（主文件 §19-3）——不跟隨 `prefers-color-scheme`，不做淺色主題：

```css
:root {
  color-scheme: dark;
  --bg: #10201f;            /* 深青墨黑底 */
  --surface: #1b2c2b;       /* 卡片/轉盤底 */
  --grid-empty: #2a3b39;    /* 未填格 */
  --grid-filled: #1c8f88;   /* 已填格底（filled 與 revealed 同色，主文件 §10）*/
  --letter-on-fill: #ffffff;
  --text: #e6ede9;
  --accent: #ff7a52;        /* 亮珊瑚：金幣、強調、選中字母 */
  --wheel-line: #ff7a52;    /* 手勢連線 */
}
```

- 已填格的青墨底白字（大字粗體）對比 ≥ 3:1，達 WCAG AA-large；完整定案色票（含陰影、次要文字等）在 `src/style.css` 的 `:root`。
- §D 的分享格盤刻意**不**跟這組變數走（用 ⬜／⬛ 的填字遊戲慣例色）——它要在別人的聊天室裡成立，跟著品牌色反而會在淺色背景失效。改色票時不用管它。
- 標題字體：系統字型堆疊（`system-ui`）先開工，品牌字體是視覺定案的事，不擋路。
